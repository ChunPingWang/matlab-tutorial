## 1. 符號運算：把手算的部分交給 MATLAB

### 歷史脈絡：從牛頓到 Mathematica

1666 年的劍橋大學因瘟疫停課，二十四歲的牛頓回到伍爾索普老家「奇蹟年」期間發明了微積分。同時間萊布尼茨在巴黎獨立發展出更易讀的記號（`dx`、`∫` 都是他發明的，至今沿用）。但**手工微積分有個致命缺陷**：對複雜函式做幾次連鎖律就會展開成幾頁紙，容易抄錯。

19 世紀的工程師為此發明了**對數表、積分表**（一本書整理 5000 條常用積分）。20 世紀電腦出現後，1960 年代 MIT 開始研發 MACSYMA — 第一個能自動做符號微積分的系統。1988 年 Stephen Wolfram 把這個概念商業化成 Mathematica，1993 年 MathWorks 推出 Symbolic Math Toolbox 把同樣能力整進 MATLAB。

### 現代工程場景

- **控制設計**：推導 PID 增益的解析表達式，知道「Kp 加倍會讓 settling time 變多少」
- **機構學**：機械手臂逆運動學，把角度反推給定座標的封閉解
- **電路分析**：用對稱結構快速簡化 Kirchhoff 方程組
- **論文寫作**：把計算過程一鍵轉成 LaTeX，附在報告裡

### MATLAB 怎麼做

```matlab
syms x          % 宣告符號變數
f = sin(x)^2 + exp(-x);
diff(f, x)      % => 2*cos(x)*sin(x) - exp(-x)
int(f, x)       % 不定積分
int(f, x, 0, inf)  % 定積分
```

### 自動解 ODE

19 世紀貝努利家族花了三代人才把彈簧質量阻尼 `m·x'' + c·x' + k·x = 0` 的所有形式手解出來。今天：

```matlab
syms x(t)
eqn = diff(x, t, 2) + 0.5*diff(x, t) + 4*x == 0;
Dx = diff(x, t);
cond = [x(0) == 1, subs(Dx, t, 0) == 0];
sol = dsolve(eqn, cond);   % 自動解出 e^(-t/4) 形式
```

![彈簧解析解](images/01_sym_spring.png)

### 泰勒展開：理解非線性近似

1715 年 Brook Taylor 發表他的展開式時可能沒想到，這個工具 200 年後會成為控制理論的命脈 — **所有「線性化」操作的本質都是泰勒展開取一階**。

控制理論中「在工作點 linearize」就是泰勒展開到一階：

![泰勒](images/01_taylor.png)

階數越高，逼近範圍越廣，但永遠在距離原點越遠處發散。這也說明為什麼**線性化控制器在「大訊號」下會崩潰** — 倒立擺擺角偏離垂直超過 ~30° 後 LQR 就失效，就是這個道理。

### 何時用符號運算

- 推導控制器增益的解析表達式
- 把複雜的微分一次算清楚
- 把結果轉成 LaTeX：`latex(sol)`

**不要**對大型矩陣或非線性 ODE 死硬用符號運算 — 會慢到吐血或根本解不出來。那是 `ode45` 的工作。

## 2. ODE 數值解：工程主力 `ode45`

### 歷史脈絡：從歐拉到二戰彈道計算

1768 年歐拉提出最早的數值解法 — **歐拉法**：把 `dy/dt = f(t, y)` 用 `y_{n+1} = y_n + h·f(t_n, y_n)` 近似。簡單但精度太差，誤差累積快。

1895 年德國數學家 **Runge** 與 1901 年的 **Kutta** 改進出「Runge-Kutta 法」— 用四個中間點的加權平均估算下一步，精度大幅提升。這就是今天 `ode45` 名字裡 **4-5 階 Runge-Kutta-Fehlberg** 的源頭。

**真正讓這套理論被工程界廣泛採用的**是二戰：1940 年代英美的彈道研究實驗室用 ENIAC 與機械式計算機算砲彈軌跡（含空氣阻力、地球自轉效應），是早期 ODE 數值求解器最大的客戶。Mary Jackson 等「電腦女郎」每天手算 RK4 迭代產出射控表。

戰後 1960s 太空競賽再次推進這套技術 — Apollo 登月導航的微分方程組就是用變步長 RK 演算法解的（之後並用到 Kalman filter，第 4 章會再講）。

### 現代工程場景

- **EV 電池熱模擬**：鋰電池熱失控的化學反應 ODE
- **無人機飛控**：6 自由度剛體動力學（位置 + 姿態）即時積分
- **流行病學**：SIR/SEIR 模型預測病毒擴散
- **藥物動力學**：藥物在血液與組織間的分布速率

### 最簡單的例子：一階衰變

放射性衰變 `dy/dt = -k·y`（這也是 1900 年居禮夫婦用來定義「半衰期」的方程式）：

```matlab
k = 0.5;
f = @(t, y) -k * y;
[t, y] = ode45(f, [0, 10], 1.0);
```

![放射性衰變](images/02_ode_decay.png)

數值解（藍）與解析解 `y = e^(-kt)`（紅虛線）重疊 — 這是檢驗 ode45 設定是否正確的最簡單 sanity check。

### 黃金規則：高階 ODE 改寫成一階狀態向量

`m·x'' + c·x' + k·x = 0` 不能直接餵 `ode45`，要改寫成：

```
令 y1 = x, y2 = x'
則 y1' = y2
   y2' = -(c/m)*y2 - (k/m)*y1
```

對應 MATLAB：

```matlab
f = @(t, y) [y(2);
             -(c/m)*y(2) - (k/m)*y(1)];

[t, Y] = ode45(f, [0 20], [1; 0]);  % 初始 x=1, x'=0
% Y(:,1) = x(t),  Y(:,2) = x'(t)
```

![彈簧 ODE](images/02_ode_spring.png)

相平面圖（右）會「往原點螺旋收斂」— 這就是「阻尼振盪」的幾何意義。這個「狀態向量」的觀念是 1960 年代 Kalman 提出狀態空間表示法後才普及的；之後第 4 章的 LQR、卡爾曼濾波都建立在這上面。

### 求解器選擇

| 求解器 | 用途 | 何時用 |
|--------|------|------|
| `ode45` | 4-5 階 Runge-Kutta-Fehlberg | 預設首選，95% 場景夠用 |
| `ode15s` | 多步隱式法（Gear's method, 1971）| 剛性 (stiff) — 化學反應、電力電子 |
| `ode89` | 8-9 階 RK | 高精度需求，例如航太軌道 |
| `ode23t` | 梯形隱式 | 中度剛性、能量守恆敏感 |

**「剛性問題」是什麼**：1952 年化學工程師 C.F. Curtiss 在求解燃燒反應時發現 — 反應裡有快變化（毫秒級）和慢變化（秒級）同時發生，顯式求解器（ode45）會被快變化逼到極小步長、跑到天荒地老。隱式求解器（ode15s）才能高效處理。

van der Pol oscillator `x'' - μ(1-x²)x' + x = 0` 在 μ 大時是經典剛性例：

![vdp](images/02_ode_vdp.png)

### 事件偵測：拋體何時落地

二戰彈道學的核心問題之一：**砲彈何時落地**？以前要解析推導，現在 `ode45` 可以在「某個量歸零」時自動停止積分：

```matlab
% hitGround.m
function [value, isterminal, direction] = hitGround(~, s)
    value = s(2);        % 監看 y
    isterminal = 1;      % 觸發即停
    direction = -1;      % 只在「下降」時觸發
end

opts = odeset('Events', @hitGround);
[t, S, te, se] = ode45(projectile, [0, 10], s0, opts);
% te = 落地時間,  se = 落地時的狀態
```

![拋體事件](images/02_ode_event.png)

工程現場用途：無人機碰地、化學反應達到指定濃度、衛星進入大氣層、彈簧壓縮到極限。

## 3. 線性代數：控制理論的根基

### 歷史脈絡：高斯的測量問題到太空導航

1809 年，**高斯**為了從 19 次望遠鏡觀測中算出小行星穀神星（Ceres）的軌道，發明了「最小二乘法」。他要解的就是一個超定的線性系統 `A·x = b`（19 個方程式、6 個未知數），這推動了現代矩陣理論的興起。

19 世紀後半 **Arthur Cayley** 與 **James Sylvester** 把「矩陣」當作獨立的數學物件來研究 — 矩陣乘法、特徵值、行列式都是這時候系統化的。

20 世紀**狀態空間理論**（1960 年代 Kalman）讓線性代數變成自動控制的母語：所有 MIMO 系統都用矩陣表示，所有設計工具（LQR、Kalman filter、卡爾曼增益）都是矩陣方程。

**Apollo 登月**：1969 年 Apollo 11 的導航電腦（AGC, 64KB RAM）裡跑的就是 Kalman filter — 用即時觀測修正「狀態向量」（位置 + 速度共 6 維），這套演算法的數學基礎就是這節介紹的工具。

### 現代工程場景

- **Google PageRank**：找最大特徵向量
- **人臉辨識**：SVD/PCA 降維
- **電力網路**：解大型稀疏線性系統（節點電壓 vs 注入電流）
- **建築結構分析**：有限元素法的本質就是解 K·u = F（剛度矩陣 × 位移 = 外力）
- **機器學習**：神經網路本質是矩陣連乘

### 解線性系統：用 `\` 不要用 `inv`

```matlab
% 解 Ax = b
x = A \ b;              % 對的
x = inv(A) * b;         % 錯的（數值差、慢）
```

`A\b` 內部用 QR 或 LU 分解，數值穩定且通常更快。**為什麼不用 inv？** 1947 年 Alan Turing 寫的「Rounding-off errors in matrix processes」一文證明：直接算 `inv(A)` 會把矩陣的「**條件數**」放大兩次，誤差爆炸；用分解法只放大一次。

### 特徵值：穩定性分析的命脈

1929 年俄裔美籍工程師 **Aleksandr Lyapunov** 把特徵值與動態系統穩定性連在一起 — 這就是今天所有控制工程系一年級教的「Lyapunov 穩定性理論」。

對狀態方程 `x' = A·x`，A 的特徵值決定一切：

| 特徵值 | 系統行為 |
|--------|---------|
| 實部全為負 | 漸近穩定（會收斂到原點） |
| 有實部為正 | 不穩定（會發散） |
| 有虛部非零 | 振盪 |
| 純虛數 | 持續等幅振盪（邊界穩定） |

```matlab
A = [0 1; -4 -0.5];
eig(A)
% => -0.25 ± 1.984i  -> 阻尼振盪
```

### 特徵向量的幾何意義

對對稱矩陣 `M = [2 1; 1 3]`，把單位圓餵進去看會被壓成什麼形狀：

![特徵向量](images/03_eigenvectors.png)

藍圈是原本的單位圓，紅橢圓是被 M 線性變換後的形狀。**特徵向量就是「不被旋轉只被縮放」的特殊方向**，縮放倍率就是對應的特徵值。

工程現場用途：找出橋樑/建築的主要振動模態。1940 年華盛頓州的 Tacoma Narrows 吊橋因風振共振而坍塌 — 事後分析就是用結構特徵值找出共振頻率 0.2Hz 剛好對上當天的風頻。今天所有高樓大廈與長跨距橋樑設計都會做這種模態分析。

### SVD：萬用的矩陣分解

1873 年 Eugenio Beltrami 首先提出，但真正成為工程工具是 1960s 後計算機普及。**SVD 是 NumPy/MATLAB 工具箱裡最被低估的瑞士刀**：

```matlab
[U, S, V] = svd(A);
% A = U * S * V'
```

`magic(8)` 看起來是普通 8x8 矩陣，但 SVD 暴露它其實只有 3 個非零奇異值 — 也就是「秩 = 3」：

![SVD](images/03_svd.png)

控制與訊號處理用 SVD 做：
- **最小平方擬合**（多餘變量時）— Netflix 推薦系統 2009 年的「Netflix Prize」冠軍演算法核心是 SVD
- **系統識別**（Hankel 矩陣降秩）— 從實驗數據推回未知 plant
- **可控/可觀測性矩陣的秩判定**（第 4 章會用）
- **影像壓縮**：JPEG 雖然用 DCT，但 SVD 是更通用的影像降維

### 條件數：別亂解病態系統

`cond(A)` 量化「A 有多接近奇異」。Hilbert 矩陣是經典病態例：

![Hilbert 條件數](images/03_hilbert_cond.png)

12x12 的 Hilbert 矩陣條件數已經達 10¹⁶ — 雙精度數值極限。意味著 `b` 的最後一位誤差會被放大到整個解都不可信。**1980 年代美國一座核電廠**就因為熱傳模擬用了病態矩陣，預測溫度與實測偏差近 50°C，後來才換成 SVD 求解。

## 4. 複數與頻域：通往控制 Bode 圖的橋

### 歷史脈絡：從傅立葉到 4G/5G

1822 年法國數學家 **Joseph Fourier** 在《熱的分析理論》中提出：**任何週期函數都能分解成正弦波的和**。當時學界譁然 — 拉格朗日公開反對，認為這違反「不連續函數沒解析表示」的直覺。但傅立葉的方法成功解開了熱傳導 PDE，無法反駁。

100 年後 **電報、無線電** 興起 — 傅立葉變換變成通訊工程的基礎工具。1928 年 Bell 實驗室的 **Harry Nyquist** 證明取樣定理（取樣率 ≥ 2 × 最高頻率才能無失真重建）。1948 年 Bell 的 **Claude Shannon** 把這套變成資訊理論。

**1965 年 Cooley & Tukey 發表 FFT** — 把 N×N 的離散傅立葉變換從 O(N²) 加速到 O(N·log N)。從此以後**所有現代通訊（4G、5G、Wi-Fi、藍牙）、所有音訊壓縮（MP3、AAC）、所有影像壓縮（JPEG）**都依賴 FFT。

控制工程方面：**1932 年 Harry Nyquist** 發表「再生理論」用複數頻域分析回授放大器穩定性。**1940 年 Hendrik Bode** 在二戰時為了設計火砲射控的自動瞄準伺服系統，提出 Bode 圖。這兩個工具是第 4 章的核心。

### 現代工程場景

- **5G 通訊**：OFDM 調變的本質就是 IFFT 編碼 + FFT 解碼
- **CT/MRI 醫學影像**：影像重建就是反 FFT
- **音樂錄音室**：Equalizer、降噪都是頻域濾波
- **車輛診斷**：發動機異常震動用 FFT 看頻譜找原因
- **地震監測**：地震波頻譜分析判斷震源距離

### Euler 公式：複指數就是旋轉

1748 年歐拉發表 `e^(jθ) = cos(θ) + j·sin(θ)`，**Richard Feynman 稱這是「整個數學最美的方程式」**。

```
e^(jθ) = cos(θ) + j·sin(θ)
```

![Euler](images/04_euler.png)

時間訊號 `e^(jωt)` 在複平面就是「以角速度 ω 旋轉的單位向量」。所有正弦訊號都可以看成兩個反向旋轉複指數的合成。**這個觀念是 Laplace 變換、傅立葉變換、Z 變換的共同根基**。

### FFT：把訊號拆成頻率

```matlab
fs = 1000;
t = (0:1/fs:1-1/fs)';
sig = 0.7*sin(2*pi*50*t) + sin(2*pi*120*t) + 0.5*randn(size(t));

Y = fft(sig);
% 取單邊頻譜
L = length(sig);
P1 = abs(Y(1:L/2+1))/L * 2;
f = fs*(0:L/2)/L;
```

![FFT](images/04_fft.png)

時域看不出來的 50Hz 和 120Hz 兩成份，FFT 一眼分離出來。

**工程應用情境**：把麥克風錄到的引擎雜音做 FFT — 若 50Hz 異常高峰，是市電耦合；若 120Hz 峰，可能是橋樑振動或泵浦氣穴；若 1kHz 寬頻雜訊，可能是齒輪磨損。整個「**機器健康診斷**」（Condition Monitoring）就是建立在 FFT 上。

### Laplace s 平面：傳遞函數的家

**1782 年 Pierre-Simon Laplace** 提出他的變換，比傅立葉早 40 年。Laplace 變換 = 「傅立葉變換 + 衰減項」，能處理「不衰減的訊號」（如階躍輸入），所以是控制理論的標準語言。

下一章控制會大量出現 `H(s) = 1/(s² + 0.5s + 4)`。在複平面 `s = σ + jω` 看 `|H(s)|`：

![Laplace 表面](images/04_laplace_surface.png)

兩個「山峰」位置正是 `s² + 0.5s + 4 = 0` 的根 — 這就是**極點**。控制工程的「極點/零點分析」全部建立在這張圖上。

### Bode magnitude = j·ω 軸切片
