# 04. 自動控制

> **這一章在解什麼問題？**
>
> 1788 年，46 歲的蘇格蘭工程師 **James Watt** 在他改良的蒸汽機上裝了一個簡單裝置 — 兩顆鐵球綁在垂直軸上、轉得越快越向外甩、把進氣閥拉小 — 蒸汽機的轉速就自動穩定下來。這個「**飛球調速器**」(centrifugal governor) 是人類歷史上第一個被大規模商業應用的回授控制系統，**直接催生了工業革命的標準化生產線**。
>
> 但飛球調速器有時候會**自激振盪** — 鐵球越甩越遠、越收越緊，整個系統劇烈震動到無法運作。1860 年代英國有工廠的調速器因此爆炸傷人。沒人知道為什麼。
>
> 1868 年，**詹姆斯 · 馬克士威**（沒錯，就是電磁學那位）發表了論文《**On Governors**》— 用微分方程證明：調速器的穩定性取決於系統矩陣的「**特徵值實部要全部為負**」。**這篇 10 頁的論文是「現代控制理論」的起點**，2 章前的線性代數工具到這裡才真正展現工程價值。
>
> 從馬克士威到今天的 240 年裡，控制理論發展出五個關鍵里程碑：
> 1. **1932 年 Nyquist** 用複數頻域判穩
> 2. **1940 年代 Bode** 把頻域分析「**從藝術變科學**」（貝爾實驗室為設計遠距電話放大器）
> 3. **1942 年 Ziegler-Nichols** PID 經驗法則
> 4. **1948 年 Walter Evans** 根軌跡（為了設計噴射機的自動駕駛）
> 5. **1960 年 Kalman** 狀態空間 + LQR + Kalman filter（**直接讓 1969 年 Apollo 登月成功**）
>
> 今天的控制工程**無處不在**：你手上的手機螢幕亮度自動調節、你車上的 ABS 防鎖死煞車、無人機懸停、SpaceX Falcon 9 一級火箭垂直回收、台積電光刻機的奈米級定位 — 全部都是這條傳承 240 年的學科在現代工程上的延伸。
>
> 這一章把這些工具串成一條完整的控制設計流程，每節都會講清楚這個工具是「**當年誰為了什麼問題發明的**」與「**今天工業界拿來幹嘛**」。

```
建立模型 (tf/ss) -> 分析性能 (step/bode/rlocus) -> 設計控制器 (pid/place/lqr) -> 驗證
```

## 1. 傳遞函數：頻域表示法

### 歷史脈絡：拉普拉斯到貝爾實驗室

1782 年法國數學家 **Pierre-Simon Laplace** 在研究機率論時發明了現在叫做「**Laplace 變換**」的工具。當時純粹是數學遊戲，沒人想到 150 年後會變成電機工程的母語。

1920 年代美國 AT&T 為了讓電話訊號跨越美國大陸（紐約到舊金山約 4000 km），需要**負回授放大器**把訊號逐段放大但又不能失真。當時的工程師赫然發現：在連續迴路裡，「**相位**」和「**頻率**」的關係決定穩定性。

1927 年貝爾實驗室的 **Harold Black** 在一艘渡輪上突然頓悟負回授放大器的原理，當場在《紐約時報》空白處寫下方程式。1932 年 **Harry Nyquist** 把這套用 Laplace 變換的傳遞函數 G(s) 系統化，從此「**傳遞函數**」成為控制工程的標準語言。

### 現代工程場景

- **音響擴大機**：所有 hi-fi 級擴大機都用負回授放大器降低失真
- **電源管理 IC**：手機晶片內的 LDO 穩壓器是 PID + 傳遞函數設計
- **馬達伺服控制**：CNC、機械手臂的精度全靠 G(s) 模型 + 控制器
- **EV 充電樁**：直流快充的電壓電流控制是 PID + 傳遞函數
- **HVAC 空調**：辦公大樓的恆溫控制全部是傳遞函數模型

從第 3 章的 RLC 我們已經看過：對線性 ODE 取 Laplace 變換，輸出/輸入比就是傳遞函數。

```matlab
% DC 馬達 (忽略電感):
% G(s) = omega(s)/V(s) = (Kt/R) / (J*s + B + Kt*Kb/R)
G = tf(Kt/R, [J, B + Kt*Kb/R]);
```

### tf-step 與 ode45 結果一致

`step(G)` 是 MATLAB 算階躍響應的一行法，比手寫 ode45 簡潔：

![馬達階躍](images/01_motor_step.png)

兩條線完全重疊，證明：**對線性系統，傳遞函數和 ODE 是等價的兩種表示**。但 `tf` 後續能直接套 `step`、`bode`、`rlocus`、`feedback`、`pidtune`，是控制工程的標準入口。

### 不同階次的響應特徵

![階次比較](images/01_order_compare.png)

- 1 階：單調上升、無過衝（例：RC 電路、單純的熱傳系統）
- 2 階欠阻尼：振盪後收斂（例：彈簧質量阻尼、RLC、汽車懸吊）
- 3 階：相位滯後更多、上升更慢（例：含致動器延遲的伺服系統）

### 極點位置 = 穩定性

![極點圖](images/01_pzmap.png)

**核心定理**（馬克士威 1868）：閉迴路所有極點實部 < 0，系統就穩定。
所有的控制設計（PID 調參、根軌跡、LQR）本質上都在「把極點推到左半平面深處」。

## 2. 時域規格：用 `stepinfo` 自動算

### 歷史脈絡：從航太控制到 ISO 標準

1940 年代美軍為了設計 P-51 野馬戰鬥機的自動駕駛，第一次系統化提出「**飛機對飛行員輸入的反應品質**」要怎麼量化 — 上升要快但不能過衝撞到飛行員、settling 要短但不能振盪。

1950 年代這些指標被寫進 MIL-STD（美軍標準）、之後被 ISO 採納成國際機械控制系統規格。今天無論是 Apple Watch 的觸控反應、特斯拉煞車踏板的線性度、或是 ASML 光刻機的台位控制，廠商都用同樣的 RiseTime / Overshoot / SettlingTime 三項規格驗收。

### 現代工程場景

- **汽車 OEM 驗收**：BMW、Toyota 對轉向系統的回應時間規格寫到毫秒級
- **工業機械手臂**：FANUC、ABB 的點到點移動時間是賣機台的關鍵 KPI
- **醫療設備**：洗腎機的流量控制器 SettlingTime ≤ 2s 是 FDA 規範
- **遊戲手把**：PS5、Xbox 控制器的力回饋響應時間直接影響玩家體驗

工程上規格通常以時域給：

| 規格 | 物理意義 |
|------|---------|
| RiseTime | 從 10% 升到 90% 的時間 |
| SettlingTime | 進入 ±2% 範圍不再出來的時間 |
| Overshoot | 超過穩態值的最大百分比 |
| Peak | 峰值 |
| PeakTime | 達到峰值的時間 |

```matlab
G = tf(wn^2, [1, 2*zeta*wn, wn^2]);
info = stepinfo(G);   % 一次拿到全部
```

### 阻尼比對響應的影響

![不同 zeta](images/02_step_zetas.png)

- ζ=0.1：明顯振盪、過衝大（例：劣質汽車懸吊壞掉時的彈跳感）
- ζ=0.7：稍微過衝、收斂快（**設計常用值** — 賓士懸吊的祕方）
- ζ=1.0：無過衝、最快無振盪（例：高級量測儀器、精密機械）

### 自然頻率 ω_n 控制速度

![不同 wn](images/02_step_wn.png)

ω_n 越大，系統反應越快（settling time 與 1/ω_n 成正比）。但實際物理系統 ω_n 受致動器頻寬限制 — 不能無限放大。**1986 年 NASA 挑戰者號太空梭發射失敗**有部分原因就是冷天 O 環硬化、彈性係數 k 變大，整個動態 ω_n 跑出設計頻寬。

## 3. 頻域：Bode、Nyquist、穩定餘裕

### 歷史脈絡：Nyquist 與 Bode 的貝爾實驗室

1932 年 **Harry Nyquist**（瑞典裔美國工程師、貝爾實驗室）發表「再生理論」— 用複數頻域分析回授放大器的穩定性。他的關鍵洞見：**把開迴路傳遞函數 G(jω) 畫在複平面上，看軌跡是否繞 -1 點**，就能判定閉迴路穩不穩。

1940 年代 **Hendrik Bode**（荷裔美國數學家、同樣貝爾實驗室）為了二戰防空火炮的自動瞄準系統需要設計頻寬大、穩定餘裕高的伺服系統，提出對數座標下畫 magnitude 與 phase。**他的論文〈Network Analysis and Feedback Amplifier Design〉(1945) 是控制工程史上引用次數最多的教科書之一**。

二戰結束後 Bode 圖被 MIT 的 Norbert Wiener 帶到「**控制論**」（Cybernetics）這個更廣的框架，影響後來 50 年的控制教育。

### 現代工程場景

- **電源 IC 設計**：所有開關電源都要看 Bode 圖確認穩定餘裕
- **音響系統**：揚聲器分頻網路的設計核心是 Bode magnitude
- **5G 基地台**：射頻放大器的線性度與穩定性用 Bode 評估
- **MEMS 感測器**：陀螺儀、加速度計的頻寬規格直接看 Bode 圖
- **醫療超音波**：探頭的頻率響應決定影像解析度

### Bode 圖

```matlab
G = tf(10, [1, 3, 2, 0]);   % 含積分器的三階 plant
bode(G);
```

![Bode](images/03_bode.png)

兩張子圖：上是 magnitude (dB)、下是 phase (deg)。橫軸 ω 取對數。

### 穩定餘裕

```matlab
margin(G);
[Gm, Pm, Wcg, Wcp] = margin(G);
```

![Margin](images/03_bode_margin.png)

- **增益餘裕 (GM)**：相位 -180° 時還能允許增益放大多少倍
- **相位餘裕 (PM)**：增益穿越 0 dB 時還有多少相位空間

工程經驗：**GM > 6 dB、PM > 30°** 是安全閾值；**PM ≈ 60°** 對應 ζ ≈ 0.6 的良好設計。**這兩個數字是 1940 年代 Bode 從上百個雷達伺服系統設計案歸納出來的**，今天還是業界寫進規格書的標準。

### Nyquist 圖

![Nyquist](images/03_nyquist.png)

**Nyquist 判穩**：把 G(jω) 在複平面上畫一圈，看軌跡是否繞 `-1` 點。對開迴路穩定的系統，**不繞 -1 點 = 閉迴路穩定**。

**Nyquist 圖在當代用途**：分析有時間延遲的系統（網路控制、PID 在無線傳輸下的穩定性）— Bode 圖看延遲不直觀、Nyquist 圖可以一眼看出延遲把軌跡向 -1 點推了多少。

## 4. 根軌跡

### 歷史脈絡：Evans 為了噴射機

1948 年美國工程師 **Walter R. Evans**（任職於洛克希德公司）為了設計新一代噴射戰鬥機的自動駕駛，需要一個「**直觀**」的工具看回授增益 K 變化時極點怎麼跑。Bode 圖好用但只看頻域、看不到極點動態。

Evans 發明了「**根軌跡**」(Root Locus) — 把閉迴路特徵方程 `1 + K·G(s) = 0` 的根當 K 變化時的軌跡畫出來。**這個方法在 1950 年代主導了美國航太工業 20 年**，是「**手算控制設計**」最快的工具（在電腦時代之前）。

雖然今天電腦可以瞬間掃出所有 K 值的響應，根軌跡仍是教科書必修 — **它培養工程師「為什麼這樣調 K 系統會變這樣」的物理直覺**。

### 現代工程場景

- **馬達調速器**：工程師看根軌跡決定速度迴路增益
- **PLL 鎖相迴路**：通訊 IC 內部的 PLL 設計仰賴根軌跡判斷捕獲頻寬
- **電力穩定器 PSS**：大型發電機的勵磁系統用根軌跡確認電網擾動時不會跳機
- **自動駕駛橫向控制**：Lane-keeping 增益調校用根軌跡確認轉向不抖動

根軌跡顯示當回授增益 K 從 0 變化到 ∞，閉迴路極點怎麼移動。

```matlab
G = tf(1, conv([1, 1], conv([1, 2], [1, 5])));
rlocus(G);
sgrid(0.7, []);   % 畫 zeta=0.7 等阻尼線
```

![根軌跡](images/04_rlocus.png)

### 標出特定 K 值的極點位置

![軌跡標記](images/04_rlocus_marks.png)

- K 小：極點靠近開迴路極點（系統慢）
- K 中：極點往右移、阻尼下降
- K 大：兩條軌跡跑到右半平面 -> 系統不穩定

### 對應的階躍響應

![不同 K 的階躍](images/04_rlocus_step.png)

K=100 時已經接近持續振盪（極點在 jω 軸附近）。

### 臨界增益

掃描 K 找閉迴路最不穩定極點的實部：

![臨界 K](images/04_rlocus_critical.png)

當實部過零的 K 值就是「臨界穩定增益」。對 G(s) = 1/[(s+1)(s+2)(s+5)] 大約是 K ≈ 129。

**工程現場用法**：找到 K_critical 後，**工業界經驗法則**是把 K 設成 K_critical / 4 ~ K_critical / 2 之間，留出充分餘裕應付負載變化與元件老化。

## 5. PID 控制：90% 工業界用的東西

### 歷史脈絡：Ziegler-Nichols 與 Elmer Sperry 的船舵

PID 的雛形可以追溯到 **1911 年美國發明家 Elmer Sperry** 為美國海軍開發的「**陀螺穩定船舵**」(gyrocompass-based autopilot)。當時的船舵控制只有比例 (P) 一項，遇到海浪會反覆過衝。Sperry 加入「**對轉舵速率的反應**」（D 項）穩定操舵 — 這是工程史上第一個 PD 控制器。

**1922 年 Nicolas Minorsky**（俄裔美國工程師）為美國海軍 New Mexico 戰艦設計航向自動控制時，注意到風與洋流會造成穩態誤差，加入「**積分**」(I 項) 修正 — **正式的 PID 結構誕生**。

**1942 年 John G. Ziegler 與 Nathaniel B. Nichols**（兩人是 Taylor Instrument 工程師）發表〈Optimum Settings for Automatic Controllers〉，提出**經驗法則調 PID 三參數**的方法 — 不需要 plant 模型、只要做兩次實驗就能調出夠用的 PID。**這篇 11 頁的工程文件是 20 世紀工業控制最常被引用的論文**。

至今全世界**約 95% 的工業控制器仍是 PID 變體** — 從電廠汽輪機、化工反應器、紙廠張力控制、到家裡的烤箱溫控、汽車冷氣，PID 無處不在。

### 現代工程場景

- **3D 列印**：印頭溫度 PID、平台溫度 PID、Z 軸 PID 都是 ME 學生第一個碰的 PID 應用
- **無人機**：每個馬達的 PID 在 100Hz~1kHz 重新計算
- **電動車快充**：電壓電流雙環 PID 確保 350kW 充電不過流
- **半導體製程**：晶圓溫度、氣體流量、腔體壓力都用 PID
- **家用咖啡機**：高階機種（La Marzocco 等）用 PID 控制 boiler 溫度到 ±0.5°C

```
u(t) = Kp*e(t) + Ki*∫e dt + Kd*de/dt
```

| 項 | 作用 | 工程口訣 |
|----|------|---------|
| P | 比例 — 反應「現在誤差多大」 | P 過小慢、過大不穩 |
| I | 積分 — 消除穩態誤差 | I 解決「永遠到不了」、過大會振盪 |
| D | 微分 — 抑制超衝、加速收斂 | D 解決「衝過頭」、會放大雜訊 |

### P / PI / PID 比較

DC 馬達速度控制：

![PID 比較](images/05_pid_compare.png)

- 純 P：有穩態誤差（系統永遠到不了目標）
- PI：誤差消失但反應慢
- PID：快但 D 太大會放大雜訊（這裡示範手調未必最優）

### 干擾抑制：閉迴路的真正價值

開迴路根本沒辦法應付負載突變。閉迴路 PID 自動修正：

![干擾抑制](images/05_pid_disturbance.png)

t=2s 時加 0.5N·m 負載擾動，PID 在約 0.5s 內把輸出拉回 1 rad/s。**這是回授控制存在的根本理由 — 馬克士威在 1868 年那篇論文的核心精神**。

### 手調 PID 的口訣

如果沒有 plant 模型，純試誤調（這是 1942 年 Ziegler-Nichols 經驗法則的簡化版）：

1. **Kd, Ki 都先設 0**，慢慢調大 Kp 直到出現穩定振盪
2. **逐步加 Ki**，直到穩態誤差消失
3. **如有過衝**，加一點 Kd 抑制
4. 反覆微調

更系統的方法是 Ziegler-Nichols 經驗法則，但對複雜 plant 不如 `pidtune` 可靠。

**工廠現場血淚教訓**：很多老師傅憑感覺亂調 PID，結果系統「能跑」但效率低 30%。今天新一代的化工/半導體廠都會用 MATLAB 的 `pidtune` 或廠商提供的自動調參工具，省下大量試誤時間。

## 6. 狀態空間：MIMO 的標準型式

### 歷史脈絡：Kalman 革命

1960 年代以前，控制理論基本上是 SISO（單輸入單輸出）的天下 — 傳遞函數、Bode、根軌跡都是 SISO 工具。**真正多輸入多輸出**的系統（飛機 6 自由度、化工廠多反應槽、電網多發電機）只能靠老師傅憑經驗一塊一塊調。

**1960 年 Rudolf E. Kalman**（匈牙利裔美國工程師、後成為加州大學的教授）在約翰霍普金斯大學提交博士論文，提出「**狀態空間**」表示法：把任何線性系統寫成 `x' = A·x + B·u`、`y = C·x + D·u` 形式。

同一年他發表了**三篇影響後世 60 年的論文**：
1. 〈A New Approach to Linear Filtering and Prediction Problems〉— **Kalman filter**
2. 〈On the General Theory of Control Systems〉— 可控性、可觀測性
3. 〈Contributions to the Theory of Optimal Control〉— **LQR**

**這套理論的最早大規模實證**：1969 年 NASA 阿波羅 11 號登月。導航電腦 AGC（64KB RAM、~1MHz CPU）就是跑簡化版 Kalman filter 持續修正太空船位置與速度。**沒有 Kalman 的理論，人類大概要再晚 10 年才能登月**。

### 現代工程場景

- **GPS + IMU 融合**：每支手機定位都靠 Kalman filter 把 GPS（精度差、頻率低）和加速度計（精度好、頻率高、會漂移）結合
- **自動駕駛**：Tesla、Waymo 的感測融合（雷達、攝影機、光達）就是 Extended Kalman Filter 變體
- **航太導航**：所有民航機的慣性導航系統 INS 用 Kalman filter
- **生物醫學**：心電圖去雜訊用 Kalman、腦電圖事件偵測用狀態空間
- **金融**：高頻交易的訊號偵測模型本質上是狀態空間

```
x' = A·x + B·u
y  = C·x + D·u
```

任何 SISO 傳遞函數都可改寫成狀態空間（不唯一），但狀態空間天生支援 MIMO（多輸入多輸出）。

```matlab
A = [0, 1; -k/m, -c/m];   % 彈簧質量阻尼
B = [0; 1/m];
C = [1, 0];               % 量測位移
D = 0;
sys = ss(A, B, C, D);
tf(sys)                   % 想看傳遞函數可以隨時轉
```

### 從 ss 跑階躍

```matlab
sys = ss(A, B, C, D);
step(sys);          % 等價於 step(tf(sys))
```

![ss step](images/06_ss_step.png)

跟第 3 章用 `ode45` 解出的彈簧質量阻尼自由響應形狀一致 — 印證「狀態空間 ↔ ODE ↔ tf」三者等價。

### 可控性 & 可觀測性

```matlab
rank(ctrb(A, B)) == size(A, 1)   % 完全可控
rank(obsv(A, C)) == size(A, 1)   % 完全可觀測
```

**可控** = 任意給定初始狀態，能透過適當的 u 在有限時間內轉到任意目標。
**可觀測** = 從輸出 y 觀察一段時間後，能推回完整狀態。

**這兩個性質是 Kalman 1960 年正式定義的**。LQR、卡爾曼濾波都假設這兩個性質。**工程實務上**：很多系統設計失敗源於沒先檢查可控/可觀測 — 例如機械手臂某些關節組合無法到達特定點、化工反應器某些變量無法從感測器推算回來。

### 極點配置 `place`

直接指定希望閉迴路極點放哪：

```matlab
desired = [-2 + 1i, -2 - 1i];
K = place(A, B, desired);
A_cl = A - B*K;
```

![極點配置](images/06_place.png)

開迴路極點 -0.25 ± 1.98i（弱阻尼）被推到 -2 ± 1i（阻尼明顯增加），閉迴路反應快很多。

### MIMO 範例

兩個質量用彈簧連接的耦合系統：

![MIMO](images/06_mimo.png)

`step(sys_mimo)` 一次畫出「每個輸入對每個輸出」的響應矩陣。對角線是「自身致動」、非對角線是「耦合效應」。

## 7. LQR：最優狀態回授

### 歷史脈絡：從 Apollo 到 SpaceX Falcon 9

PID 適合 SISO；對 MIMO 或多狀態問題，**LQR**（Linear Quadratic Regulator）是首選。**Kalman 1960 年那篇〈Contributions to the Theory of Optimal Control〉就是 LQR 的原始論文**。

LQR 的理論優美但在 1960 年代電腦算力不足時受限。**讓 LQR 真正普及的是 1969-1972 年的 Apollo 計劃** — Draper 實驗室（MIT 的 Charles Stark Draper）為阿波羅登月艙設計姿態控制與下降軌跡，是首次大規模實機驗證 LQR 在嚴苛環境下的可靠性。

**今天最戲劇化的 LQR 應用是 SpaceX Falcon 9 一級火箭垂直回收** — 從 80km 高空、5 倍音速的姿態切換到精準垂直降落，全程的 9 引擎差動推力控制就是 LQR 變體（實際是 MPC，模型預測控制，LQR 的延伸）。**這個任務的數學嚴格程度遠超 Apollo**，但理論基礎是同一條 Kalman 1960 年的論文。

### 現代工程場景

- **無人機懸停**：DJI、Skydio 的姿態控制核心是 LQR 或其延伸
- **電動機車**：Ride-by-wire 的橫向穩定控制
- **電動人造心臟**：Heartmate 等心臟輔助裝置的轉速控制
- **半導體曝光機**：ASML EUV 的鏡片陣列定位（5nm 精度）用 LQR
- **核融合反應器**：ITER 的等離子體形狀控制研究中用 LQR
- **複雜化工廠**：BASF、Dow Chemical 的多反應槽協同控制用 MPC（LQR 延伸）

**最優化問題**：

```
最小化 J = ∫₀^∞ (x'·Q·x + u'·R·u) dt
```

- Q 大：罰狀態偏離（要快收）
- R 大：罰控制能量（致動器有限制）

```matlab
K = lqr(A, B, Q, R);    % 一行得到最優增益
```

### 倒立擺：經典 LQR 試金石

倒立擺向上的平衡點是不穩定的（小擾動就倒）。線性化後 A 矩陣有正實部特徵值：

```
開迴路極點：
   0        <- 車位置自由
  -0.14     <- 車有摩擦
  -5.61     <- 擺收斂模態
   5.57     <- 擺發散模態（這就是不穩定）
```

倒立擺是控制工程的「**Hello World**」 — 因為它**就是 Falcon 9 火箭直立回收的簡化版**！火箭推進器在底下、整個火箭比擺長很多、要對抗風與燃料晃動，本質就是「**會晃的倒立擺**」。

設 `Q = diag(10, 1, 100, 1)`（強調擺角不能歪）、`R = 0.1`：

![LQR 響應](images/07_lqr_response.png)

從擺角 5.7°、車位置 0 開始，LQR 在 ~3 秒內把擺收斂回 0、車回到原點。

### 控制力

![控制力](images/07_lqr_control_effort.png)

擾動越大、Q 越大，需要的控制力 |u| 就越大。**SpaceX 的火箭引擎控制就是這個概念的放大版** — 燃料是有限資源（R 對應燃料消耗），但又要精準降落（Q 對應位置誤差），兩者權衡決定整個任務成功率。

### Q 與 R 的權衡

固定 Q，掃 R：

![Q-R 權衡](images/07_lqr_R_tradeoff.png)

- R 小：可以用很大的控制力 -> 收斂快（火箭多噴點燃料）
- R 大：限制控制力 -> 收斂慢但實作可行（省燃料但容錯空間小）

LQR 的設計直覺：**Q 是「結果優先級」、R 是「資源預算」**。

### LQR 的優勢與限制

優點：
- 一行算出全部增益 K
- 自動保證閉迴路穩定（A-BK 所有極點實部 < 0）
- 對 MIMO 自然延伸
- **有穩定餘裕保證**（PM ≥ 60°, GM = ∞）— 這是 Kalman 1964 年證明的，工程界稱為「**LQR 的 60° 定理**」

限制：
- 假設**完整狀態可量測** — 實際常需要狀態觀測器（Luenberger 或 Kalman filter）
- 是**線性化模型的最優**，大訊號 / 非線性效應不在考慮中
- Q、R 還是要憑工程感覺挑

**從 LQR 進一步的方向**（這些是今天博士研究與業界產品的前沿）：
- **MPC（Model Predictive Control）**：LQR 的滾動窗口版本，可處理約束（電壓上限、機構行程）— Tesla、Waymo 自駕用
- **強化學習控制**：DeepMind、OpenAI 嘗試用 RL 取代 LQR — 適合模型未知的系統
- **Adaptive Control**：對時變系統（老化的引擎、變化的負載）自適應調參
- **Robust Control（H∞）**：對模型不確定性的最壞情況最優化
